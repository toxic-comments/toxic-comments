## Запуск приложения

Все команды командной строки выполняются в текущей директории.

Для локального запуска приложения нужно убедиться, что в системе работает docker compose, для этого нужно выполнить:
```shell
docker compose version
```
Если выводится версия значит все в порядке.

Также нужно установить все зависимости python, лучше делать это в виртуальном окружении:

```shell
pip install -r requirements.txt
```

```shell

```

Выполнить локальный запуск можно двумя способами: 

### Поднять базу данных в контейнере, а приложение запустить на хосте
Удобно для разработки и быстрого теста

1. Скачать образ postgres и запустить контейнер с базой:
```shell
docker compose up -d db
```

2. Применить миграции базы данных (если они не были применены ранее, проверится автоматически)
```shell
alembic upgrade head 
```

3. Запустить приложение в dev режиме:
```shell
fastapi dev app/main.py
```

Приложение запустится на http://localhost:8000, по адресу http://localhost:8000/docs будет доступна swagger-документация,
там можно потыкать какие есть http-ручки. fastapi умеет динамически обновлять файлы приложения, поэтому если что-то
изменить в коде, изменения сразу будут работать без перезапуска приложения.

Postgres будет доступна по адресу http://localhost:5432, для локального запуска логин и пароль будут postgres/postres.
База данных называется toxic-comments. После применения миграций там будет две таблицы: `alembic` - служебная для алембика
и `forward_call` - туда сохраняются вызовы метода POST /forward.

### Поднять базу и приложение в раздельных контейнерах

1. Скачать образ postgres и собрать образ с приложением, запустить контейнеры:
```shell
docker compose up
```

Приложение запустится на http://localhost:80, по адресу  http://localhost:80/docs также будет доступна документация.
